Getting Started with Alexa Skills Kit

written 2015-09-12

The Alexa Skills Kit is a platform that allows developers to create content for Amazon Echo, specifically the Amazon Alexa Virtual Assistant. The platform is in its early stages (and quite limitied), but it still allows for some neat projects.

Brief Overview

Getting a skill to run on your Echo is fairly straightforward, and I highly recommend using Amazon's provided resources to simplify the process. In order to create a skill, you first need to go to the Amazon Developers Dashboard and set one up, theres a bit of information you'll need to determine what it does. The easiest way to create responses is using AWS Lambda, which is configured to handle Alexa requests. The Lambda function will also allow you to send requests to other services you may want to integrate with.

Skill Setup

Creating a skill requires two key pieces of information: intents and utterances. Intents tell your program what function the user is trying to perform, such as: turn on the lights, read my email, or tell a joke. When your AWS Lambda function runs, the request will indicate the detected intent. Utterances are the phrases and information you need for each intent.

An example of all of this could be a skill that tells you the weather at a given location. In this case you'd need to recognize that the user is looking for the weather, determine what location they say, and then give a response. This would roughly look like:

Intent: "WeatherIntent" - Define an intent for the user wanting the weather.
Slots: "Location" - We need to pull out the location the user says.
Utterance 1: "WeatherIntent What is the weather in {oregon|Location}" - This will indicate that if the user says this phrase our program will recieve a weather intent and a location of oregon.

Amazon has unfortunately designed utterances to be too specific, often requiring you to type out every way something could be said. They have recently put out a way to create lists (of states for example) to help generate more without seeming repetitive, but this schema does create some restrictions.

Besides using the list tool, there is another way to simplify the process. By abusing the utterance a little you can force the Alexa Skill Engine to give your program the full phrase the user said, allowing you to programatically parse the input. It would be much simpler to just look for the word weather and a location than define everything before. In order to do this, just define some dummy utterances with multiple words set to a variable that will contain the full string. For example:

Utterance 1: "WeatherIntent {word|UserPhrase}"
Utterance 2: "WeatherIntent {a phrase|UserPhrase}"
Utterance 3: "WeatherIntent {some more words|UserPhrase}"

Having these variable length slot-matching commands in it just ends up making the engine give you the full string in the UserPhrase variable.

Creating a Lambda

AWS Lambda allows for creating a function in Java, Node.js, and (recently) Python 2.7. I recommend creating the skill in Node, as it is easy enough to make requests to outside services and Amazon has a library for creating responses that makes it really easy. A good tip: don't use the library that Amazon provides in the Alexa tutorial, use the library that their engineers created for the skills they share. It's much simpler, and allows you to use functions like tell() and tellWithCard() for responses. No JSON formatting required!

Alexa Speak

The text that Alexa returns is not very useful for parsing. Individual letters, for example W, are always followed by a period and numbers are always written in word form. Additionally, the speech recognition is unreliable (especially compared to Google). I highly recommend creating an endpoint for translating some of this back in to English, as well as converting frequently mis-heard words to your project's domain-specific vocabulary. This really only matters when you don't use the pre-defined utterances.

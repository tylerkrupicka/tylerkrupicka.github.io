Staticky Version 2

written 2014-11-26

Today I made some major updates to the staticky site generator that my site runs on, to allow for a wide variety of sites to be created with it. Previously, the generator was tailored to providing only the functionality that this site needed, without allowing much creativity in site layout.

On Github

Setup

The first new feature I added is a setup routine, which only runs in a near empty directory. This function automatically sets up base folders and files in the project, so that everything is in the correct place and named correctly. Now creating a functioning (blank) site takes seconds, which will greatly simplify creating other sites populated by staticky.

Elements

A major flaw of Staticky 1 was that it only replaced variables, not full site elements. I wanted the ability to insert full elements (such as nav bars or footers) in to layouts, so I created an elements directory that allows that.

Layout Overhaul

Previously there was a set number of layouts allowed by the generator, which literally only allowed for sites mirroring my sites functionality. This is no longer the case. Any layout created will be treated as its own page, and will work and propigate with any name.

Picture Thumbnails

While my site uses text-based post thumbnails, I realized that many sites would probably include a picture element. I made a minor tweak so that now all posts have a picture thumbnail field.

Create a Content-Based Site in Seconds

All of these updates bring the script a lot closer to my goal of creating a multi-purpose site generator that is extremely simple to set up and use. I hope to use the generator to quickly create and propigate sites for other projects.

Check out the Github page to try it out.

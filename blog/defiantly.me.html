defiantly.me

written 2015-01-01

After the @DefiantLee_ twitter bot had its API key taken away, I decided to create a Django web app to utilize my machine learning classifier. The site is hosted on digitalocean, and uses the twitter api to display and store recent misuses.

defiantly.me

Django

I'm using Django 1.7, python 2.7, tweepy, and nltk to operate the website. The site polls for tweets every time the page is loaded, and saves new tweets to the sqlite database. I originally had a lot of difficulty setting up the Naive Bayes classifier to initialize when the site started, because Django doesn't really have a good system for that. I only want to initialize the classifier (which is loading thousands of tweets) once when the site starts, because creating a new one per request would take an insane amount of time and resources. I ended up creating the classifier in the Django settings.py file, which is only run once.

Gunicorn / Nginx

Digital Ocean uses Gunicorn and Nginx to operate Django applications on their servers. Gunicorn automatically creates multiple instances of the Django app, so that it can easily handle multiple requests to the site. Learning some of the configurations for these utilities, as well as modifying file permissions, took some time.

Twitter Widgets

While twitter has an api for creating embedded tweets, as I was reading the documentation I noticed they had restrictions on usage and volume. I decided to avoid authentication and twitter errors entirely, and just use their "static" embed widget html (the embed code they give you when you click embed on a tweet). This seemed to work fine, although it's not very fast loading.
